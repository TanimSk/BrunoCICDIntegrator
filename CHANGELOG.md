## Technical Summary:

This project is a Python-based backend web application using Django 5.2 with Django REST Framework (DRF) for building REST APIs. It implements JWT-based authentication and user management enhanced with `dj-rest-auth` and `django-allauth` for registration, login (including Google OAuth), password reset, and email confirmation flows.

The core of the project lies in the `server/` folder, which is the Django project root containing project settings, URLs, ASGI & WSGI configuration, and multiple Django apps:

- **administrator/**: Manages custom user models extending Django's AbstractUser with role flags (`is_admin`, `is_customer`), custom authentication views (`LoginWthPermission`), password change, serializers with custom exception handling, and authorization logic. It also includes migrations and admin interfaces.

- **food/**: Contains a simple food catalog app with models (`Food`), serializers, and REST API views implementing full CRUD with search, filtering, ordering, and pagination. Permissions are mostly open (`AllowAny`) which suggests this app is a public-facing API endpoint.

- **all_auth_extended/**: Contains customization for django-allauth email flow, rewriting email confirmation and password reset URLs to point to frontend routes, facilitating SPA frontend integration.

- **utils/**: Holds shared utilities for the project such as a standard paginated response class (`StandardResultsSetPagination`), and a threaded HTML email sender supporting attachments (images and PDFs generated by WeasyPrint).

- **templates/account/email/**: Overrides the default django-allauth email templates for confirmation and password reset to customize emails sent to users.

The backend supports social authentication via Google OAuth2 integrated with dj-rest-auth and django-allauth, exposing endpoints like `/rest-auth/google/`. JWT tokens are central to authentication with refresh and verify endpoints available.

The project is configured with development settings allowing all CORS origins and debug enabled, but shows comments for production readiness, including environment-based secret keys and stricter host and CORS lists.

Database setups include SQLite by default with explicit placeholders for PostgreSQL, and migrations are present for apps, indicating use of Django’s ORM.

Additional dependencies include:
- Channels + Daphne + Redis stack for WebSocket support and async capabilities.
- Cloudinary for media handling.
- WeasyPrint for PDF generation in email attachments.

Scripts and maintenance tools such as a shell script to remove migration files (`remove_migration_files.sh`) exist to help manage local dev environment state.

Overall, this is a modern Django/DRF backend boilerplate with comprehensive auth features, custom email handling, social logins, JWT support, and a simple API for managing food items, designed for integration with a frontend SPA (react, vue, angular implied but not in repo).
